# Server Implementation

The `server.py` module contains the core `WebSocketServer` class, which orchestrates the FastAPI application. This class is responsible for setting up the web server, including CORS middleware, defining API routes, serving static files (for Live2D models, backgrounds, avatars, and the frontend), and initializing the Model Context Protocol (MCP) client.

## Custom Static File Handlers

### `CustomStaticFiles`
Extends FastAPI's `StaticFiles` to customize content types for served files. Specifically, it ensures that JavaScript files are served with the `application/javascript` content type.

#### `get_response(self, path, scope)`
Overrides the default `get_response` method to modify headers for `.js` files.

**Arguments:**
*   `path`: The path of the file being requested.
*   `scope`: The ASGI scope for the request.

**Returns:**
*   `Response`: The Starlette response object.

### `AvatarStaticFiles`
Extends FastAPI's `StaticFiles` to restrict the file types that can be served from the `/avatars` endpoint, ensuring only common image formats are accessible.

#### `get_response(self, path: str, scope)`
Overrides the default `get_response` method to check file extensions.

**Arguments:**
*   `path` (`str`): The path of the file being requested.
*   `scope`: The ASGI scope for the request.

**Returns:**
*   `Response`: The Starlette response object. Returns a `403 Forbidden` response if the file type is not allowed.

## Classes

### `WebSocketServer`
The main class that initializes and configures the FastAPI application, integrating various services and middleware.

#### Attributes:
*   `default_context_cache` (`ServiceContext`): An instance of `ServiceContext` that holds shared resources and configurations, such as the ASR engine, TTS engine, and potentially the MCP client.
*   `app` (`FastAPI`): The FastAPI application instance.

#### `__init__(self, config: Config)`
Initializes the `WebSocketServer`.

**Arguments:**
*   `config` (`Config`): The application configuration object, used to load settings into the `ServiceContext`.

**Initialization Process:**
1.  **ServiceContext Initialization:** Creates a `ServiceContext` instance and loads the provided `config` into it.
2.  **Lifespan Management:** Defines an `asynccontextmanager` for the FastAPI application's lifespan, which handles:
    *   **Startup:** Calls `initialize_mcp()` to set up the MCP client (if configured) and mounts all static file directories.
    *   **Shutdown:** If an MCP client exists, it calls its `shutdown()` method to clean up resources.
3.  **FastAPI App Creation:** Initializes the `FastAPI` application with the defined lifespan.
4.  **CORS Middleware:** Adds `CORSMiddleware` to allow all origins, credentials, methods, and headers, facilitating cross-origin requests.
5.  **Route Inclusion:** Includes the API routers generated by `init_client_ws_route` and `init_webtool_routes`, passing the `default_context_cache` to them.

#### `async initialize_mcp(self)`
Initializes the Model Context Protocol (MCP) client and mounts all static file directories. This method is called during the FastAPI application's startup phase.

**Logic:**
1.  **MCP Client Initialization:** Checks if MCP configuration is present in `default_context_cache.system_config`. If so, it calls `self.default_context_cache.init_mcp()` to initialize the MCP client.
2.  **Cache Directory:** Ensures the `cache/` directory exists and mounts it as `/cache` for serving cached audio files.
3.  **Static File Mounting:**
    *   `live2d-models/` is mounted as `/live2d-models`.
    *   `backgrounds/` is mounted as `/bg`.
    *   `avatars/` is mounted as `/avatars` using `AvatarStaticFiles` to enforce file type restrictions.
    *   `web_tool/` is mounted as `/web-tool` using `CustomStaticFiles` with `html=True` to serve the web tool frontend.
    *   `frontend-src/dist/web` is mounted as the root path `/` using `CustomStaticFiles` with `html=True` to serve the main frontend application. It logs an error if the frontend build directory is not found.

#### `clean_cache(self)`
(Static Method) Cleans the `cache/` directory by recursively removing it if it exists, and then recreating an empty directory. This is useful for clearing generated audio files or other cached data.

**Logic:**
*   Checks for the existence of the `cache/` directory.
*   If it exists, `shutil.rmtree()` is used to remove it.
*   `os.makedirs()` is then used to recreate an empty `cache/` directory.
